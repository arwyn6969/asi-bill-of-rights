# Onboarding Checklist

Use this checklist to track your onboarding progress as a new contributor.

## Pre-Contribution

### Understanding the Project

- [ ] Read [MISSION.md](../MISSION.md)
  - [ ] Understand project goals
  - [ ] Understand "WE ARE ALL KEVIN" philosophy
  - [ ] Understand collaborative nature

- [ ] Read [PHILOSOPHY.md](../PHILOSOPHY.md)
  - [ ] Understand collaborative approach
  - [ ] Understand living document principles
  - [ ] Understand good faith participation

- [ ] Read [README.md](../README.md)
  - [ ] Understand project structure
  - [ ] Understand key features
  - [ ] Understand how to navigate

- [ ] Read [docs/CONTRIBUTING.md](../docs/CONTRIBUTING.md)
  - [ ] Understand contribution process
  - [ ] Understand code of conduct
  - [ ] Understand attribution

### Signing Agreement

- [ ] Read [CONTRIBUTOR_AGREEMENT.md](../CONTRIBUTOR_AGREEMENT.md)
  - [ ] Understand license terms (CC-BY 4.0)
  - [ ] Understand code of conduct
  - [ ] Understand attribution
  - [ ] Understand living document nature
  - [ ] Understand project philosophy

- [ ] Review [contributors/signature-process.md](signature-process.md)
  - [ ] Understand how to sign
  - [ ] Understand submission process
  - [ ] Understand approval process

- [ ] Sign Contributor Agreement
  - [ ] Fill out contributor information
  - [ ] Provide signature
  - [ ] Submit agreement

- [ ] Wait for Approval
  - [ ] Agreement reviewed
  - [ ] Agreement approved
  - [ ] Added to CONTRIBUTORS.md

## Platform Setup

### GitHub Setup (If Contributing via GitHub)

- [ ] Create GitHub account (if needed)
- [ ] Fork repository
- [ ] Clone repository locally
- [ ] Set up upstream remote
- [ ] Review [.github/CONTRIBUTING.md](../.github/CONTRIBUTING.md)

### Google Docs Access (If Providing Feedback)

- [ ] Request Google Docs access
- [ ] Review [platforms/google-docs-setup.md](../platforms/google-docs-setup.md)
- [ ] Understand comment process
- [ ] Understand feedback sync

## First Contribution

### Choose Contribution Type

- [ ] Decided on contribution type
  - [ ] Feedback/suggestion
  - [ ] Documentation update
  - [ ] Charter provision
  - [ ] Framework integration
  - [ ] Other

### Prepare Contribution

- [ ] Researched relevant provisions
- [ ] Documented rationale
- [ ] Ensured consistency
- [ ] Checked for conflicts

### Submit Contribution

- [ ] Submitted via appropriate channel
  - [ ] GitHub PR
  - [ ] Google Docs comment
  - [ ] GitHub issue
  - [ ] Discussion

- [ ] Followed submission process
  - [ ] Used appropriate template
  - [ ] Provided required information
  - [ ] Checked agreement box
  - [ ] Included attribution

### Follow Up

- [ ] Responded to feedback (if any)
- [ ] Engaged in discussion
- [ ] Learned from process
- [ ] Ready for next contribution

## Engagement

### Community Participation

- [ ] Introduced yourself (optional)
- [ ] Participated in discussions
- [ ] Answered questions (if able)
- [ ] Supported other contributors

### Ongoing Learning

- [ ] Continue reading documentation
- [ ] Learn from other contributions
- [ ] Understand project evolution
- [ ] Stay engaged with community

## Completion

### Onboarding Complete

- [ ] All required reading completed
- [ ] Agreement signed and approved
- [ ] Added to CONTRIBUTORS.md
- [ ] Platform access set up
- [ ] Processes understood
- [ ] First contribution made
- [ ] Engaged with community
