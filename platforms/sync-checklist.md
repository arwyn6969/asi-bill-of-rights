# Sync Checklist

Use this checklist for each sync operation between GitHub and Google Docs.

## Pre-Sync Checklist

### Preparation

- [ ] Review GitHub changes since last sync
- [ ] Check CHANGELOG.md for documented changes
- [ ] Verify version numbers match
- [ ] Review recent commits
- [ ] Identify all changes to sync

### Backup

- [ ] Create backup copy of current Google Docs
- [ ] Archive old version with date
- [ ] Verify backup successful
- [ ] Document backup location

### Documentation Review

- [ ] Review sync-procedure.md
- [ ] Check sync-strategy.md
- [ ] Review version-control.md
- [ ] Note any special considerations

## Sync Execution Checklist

### Content Sync

- [ ] Copy content from GitHub (charter/asi-bor-vX.X.md)
- [ ] Paste into Google Docs
- [ ] Verify all content copied
- [ ] Check for missing sections

### Formatting

- [ ] Apply appropriate heading styles
- [ ] Format lists correctly
- [ ] Check table formatting (if any)
- [ ] Verify code blocks (if any)
- [ ] Check cross-references formatting

### Metadata Update

- [ ] Update version number in header
- [ ] Update date in header
- [ ] Update "Last Sync" date
- [ ] Update GitHub repository link
- [ ] Verify all metadata accurate

### Links and References

- [ ] Verify all GitHub links work
- [ ] Check external links
- [ ] Verify cross-references
- [ ] Test all links

### Version Management

- [ ] Archive old Google Docs version
- [ ] Name archive with version and date
- [ ] Move to archive folder
- [ ] Update main document title
- [ ] Verify version alignment

## Post-Sync Checklist

### Verification

- [ ] Verify content matches GitHub
- [ ] Check version numbers match
- [ ] Verify formatting is correct
- [ ] Test all links
- [ ] Check cross-references

### Documentation

- [ ] Document sync date
- [ ] Note changes synced
- [ ] Update sync log
- [ ] Record any issues
- [ ] Document resolutions

### Notification

- [ ] Notify moderators of sync
- [ ] Update contributors if needed
- [ ] Post in community channels (if applicable)
- [ ] Update project status

### Cleanup

- [ ] Resolve old comments (if applicable)
- [ ] Clean up working documents
- [ ] Organize archive
- [ ] Update documentation

## Feedback Collection Checklist

### Comment Review

- [ ] Review all Google Docs comments
- [ ] Identify actionable feedback
- [ ] Note non-actionable comments
- [ ] Group related comments

### Issue Creation

- [ ] Create GitHub issues for actionable feedback
- [ ] Use feedback template
- [ ] Link to Google Docs comments
- [ ] Tag appropriately
- [ ] Assign if needed

### Comment Resolution

- [ ] Respond to comments in Google Docs
- [ ] Reference GitHub issues
- [ ] Thank contributors
- [ ] Resolve comments
- [ ] Document resolutions

## Quality Assurance Checklist

### Content Quality

- [ ] No content lost
- [ ] No formatting errors
- [ ] No broken links
- [ ] No version mismatches
- [ ] All sections present

### Process Quality

- [ ] Followed procedures
- [ ] Documented properly
- [ ] Notified appropriately
- [ ] Verified successfully
- [ ] No errors introduced

## Troubleshooting Checklist

### If Issues Found

- [ ] Identify issue type
- [ ] Check documentation
- [ ] Review recent changes
- [ ] Determine solution
- [ ] Implement fix
- [ ] Document issue and resolution
- [ ] Update procedures if needed

## Sync Completion

### Final Steps

- [ ] All checklists completed
- [ ] Sync verified successful
- [ ] Documentation updated
- [ ] Notifications sent
- [ ] Ready for next sync
