# Google Docs Setup Guide

This guide explains how to set up the Google Docs version of the ASI Bill of Rights charter for collaborative editing and community feedback.

## Overview

The Google Docs version serves as:
- **Collaborative Editing Space**: For moderators to work together
- **Community Feedback Platform**: For community members to comment
- **Accessible Alternative**: For those who prefer Google Docs over GitHub
- **Comment Collection**: For gathering community input

## Setup Steps

### Step 1: Create Google Docs Document

1. **Create New Document**
   - Go to Google Docs
   - Create new document
   - Title: "ASI Bill of Rights - Draft 4.0"
   - Add subtitle with date and version

2. **Copy Content**
   - Copy content from `charter/asi-bor-v4.0.md`
   - Paste into Google Docs
   - Format appropriately (headings, lists, etc.)

3. **Add Metadata**
   - Document version in header
   - Link to GitHub repository
   - Link to project website (if available)
   - Attribution information

### Step 2: Set Up Sharing Permissions

1. **Click Share Button**
   - Click "Share" in Google Docs
   - Configure access settings

2. **Access Levels**
   - **Moderators**: Editor access
   - **Contributors**: Commenter access
   - **Public**: Viewer or Commenter access (as needed)

3. **Specific People**
   - Add moderators as editors
   - Add contributors as commenters
   - Configure public access as appropriate

### Step 3: Configure Document Settings

1. **Version History**
   - Enable version history
   - Set up naming convention for versions
   - Document major changes

2. **Comment Settings**
   - Enable comments
   - Configure notifications
   - Set up comment resolution process

3. **Suggestion Mode**
   - Use for collaborative editing
   - Review suggestions before accepting
   - Track who made what changes

### Step 4: Link to GitHub

1. **Add GitHub Link**
   - Include link in document header
   - Link to repository
   - Link to specific version

2. **Sync Instructions**
   - Add note about sync process
   - Link to sync documentation
   - Explain source of truth

### Step 5: Create Supporting Documents

1. **Comment-Only Version**
   - Create copy for public comment
   - Set to comment-only access
   - Link from main document

2. **Working Documents**
   - Create drafts for collaborative editing
   - Name clearly (e.g., "Draft 4.1 - Working")
   - Archive when done

## Document Structure

## Access Management

### Editor Access (Moderators)

**Who Gets Editor Access**:
- Human moderators
- Co-founding moderators (if applicable)
- Project administrators

**Responsibilities**:
- Make edits to document
- Review and accept suggestions
- Resolve comments
- Manage versions

### Commenter Access (Contributors)

**Who Gets Comment Access**:
- Signed contributors
- Community members
- Interested parties

**Responsibilities**:
- Provide feedback via comments
- Suggest improvements
- Ask questions
- Engage in discussions

### Viewer Access (Public)

**Who Gets Viewer Access**:
- General public
- Researchers
- Policy makers

**Capabilities**:
- View document
- Read comments (if public)
- Cannot edit or comment

## Version Control

### Naming Conventions

- **Main Document**: "ASI Bill of Rights - Draft 4.0"
- **Working Documents**: "Draft 4.1 - Working - [Date]"
- **Archived Versions**: "Draft 4.0 - Archived - [Date]"

### Version History

1. **Use Version History Feature**
   - Name versions at major changes
   - Document what changed
   - Link to GitHub commits

2. **Archive Old Versions**
   - Move to archive folder
   - Keep for reference
   - Link from main document

## Comment Management

### Comment Process

1. **Receive Comments**: Comments appear in document
2. **Review Comments**: Moderators review
3. **Respond**: Respond to comments
4. **Resolve**: Resolve when addressed
5. **Sync**: Sync relevant comments to GitHub

### Comment Guidelines

- **Be Constructive**: Provide helpful feedback
- **Be Specific**: Point to exact sections
- **Be Respectful**: Maintain "WE ARE ALL KEVIN" spirit
- **Be Clear**: Explain your perspective
