# Contributor Journey Map

This document maps the journey of contributors from first contact to active participation in the ASI Bill of Rights project.

## Purpose

This journey map helps:
- Understand the contributor experience
- Identify onboarding steps
- Clarify processes and expectations
- Support new contributors
- Improve the contribution experience

## Journey Stages

### Stage 1: Discovery

**What Happens**:
- Contributor discovers the project
- Reads README.md
- Reviews project philosophy
- Understands "WE ARE ALL KEVIN" approach

**Resources**:
- `/README.md` - Project overview
- `/docs/MISSION.md` - Mission statement
- `/docs/PHILOSOPHY.md` - Collaborative philosophy
- `/charter/asi-bor-v5.0.md` - Main charter document (current)

**Outcome**: Contributor understands project purpose and philosophy

### Stage 2: Initial Interest

**What Happens**:
- Contributor reviews charter content
- Explores documentation
- Checks contribution guidelines
- Considers how to contribute

**Resources**:
- `/docs/CONTRIBUTING.md` - Contribution guidelines
- `/docs/IMPLEMENTATION.md` - Implementation guide
- `/charter/` - Charter documents
- `/schemas/` - Machine-readable formats

**Outcome**: Contributor identifies potential contribution areas

### Stage 3: Agreement Signing (Required)

**What Happens**:
- Contributor reads Contributor Agreement
- Understands terms and conditions
- Signs agreement (digital or typed)
- Submits signed agreement

**Resources**:
- `/contributors/CONTRIBUTOR_AGREEMENT.md` - Full agreement
- `/contributors/agreement-template.md` - Template
- `/contributors/signature-process.md` - Signing process
- `/contributors/signature-submission.md` - Submission process

**Steps**:
1. Read `/contributors/CONTRIBUTOR_AGREEMENT.md`
2. Review agreement terms
3. Fill out contributor information
4. Sign agreement
5. Submit via `/contributors/signature-submission.md`

**Outcome**: Contributor agreement signed and submitted

### Stage 4: Onboarding

**What Happens**:
- Contributor completes onboarding checklist
- Reviews governance structure
- Understands contribution process
- Sets up development environment

**Resources**:
- `/contributors/ONBOARDING.md` - Onboarding guide
- `/contributors/onboarding-checklist.md` - Checklist
- `/governance/GOVERNANCE.md` - Governance structure
- `/governance/roles.md` - Role definitions

**Steps**:
1. Complete onboarding checklist
2. Review governance documents
3. Understand roles and responsibilities
4. Set up GitHub access (if contributing code/docs)
5. Review contribution types

**Outcome**: Contributor ready to contribute

### Stage 5: First Contribution

**What Happens**:
- Contributor identifies contribution type
- Prepares contribution
- Follows contribution guidelines
- Submits contribution

**Resources**:
- `/docs/CONTRIBUTING.md` - Contribution guidelines
- `/docs/TERMINOLOGY.md` - Terminology standards
- `/docs/CROSS-REFERENCE-INDEX.md` - Cross-reference guide
- Issue templates for different contribution types

**Contribution Types**:
1. **Small Changes**: Direct pull request or issue
2. **Medium Changes**: Open issue first, then pull request
3. **Major Changes**: Discussion issue, then amendment process

**Steps**:
1. Identify contribution type
2. Review relevant documentation
3. Prepare contribution following guidelines
4. Submit via appropriate channel
5. Wait for review

**Outcome**: First contribution submitted

### Stage 6: Review Process

**What Happens**:
- AI moderator reviews contribution
- Human moderator reviews contribution
- Feedback provided (if needed)
- Contributor revises (if needed)
- Contribution approved or rejected

**Resources**:
- `/governance/moderation-guidelines.md` - Moderation standards
- `/governance/decision-process.md` - Decision process
- `/governance/GOVERNANCE-FLOWCHART.md` - Process flowcharts

**Review Stages**:
1. **AI Moderator Review**: Technical and principle validation
2. **Human Moderator Review**: Strategic and final validation
3. **Community Review** (for major changes): Community input

**Outcome**: Contribution reviewed and decision made

### Stage 7: Integration

**What Happens**:
- Contribution integrated into project
- Attribution recorded
- Documentation updated
- Contributor recognized

**Resources**:
- `/CONTRIBUTORS.md` - Contributor list
- `/contributions/contributions.json` - Contribution log
- `/docs/CHANGELOG.md` - Change log

**Integration Steps**:
1. Contribution merged/integrated
2. Attribution added to CONTRIBUTORS.md
3. Entry added to contributions.json
4. Documentation updated (if needed)
5. Changelog updated (if applicable)

**Outcome**: Contribution integrated and attributed

### Stage 8: Ongoing Participation

**What Happens**:
- Contributor continues contributing
- Engages in community discussions
- Reviews other contributions
- Supports new contributors

**Resources**:
- `/docs/CONTRIBUTING.md` - Ongoing contribution guidelines
- `/governance/` - Governance resources
- Community discussions and issues

**Ongoing Activities**:
- Regular contributions
- Community engagement
- Feedback and reviews
- Process improvement suggestions

**Outcome**: Active contributor status

## Contributor Paths

### Path A: Documentation Contributor

**Focus**: Documentation improvements

**Journey**:
1. Discovery → Initial Interest
2. Agreement Signing
3. Onboarding
4. First Contribution (documentation fix)
5. Review Process
6. Integration
7. Ongoing Participation (documentation improvements)

### Path B: Charter Contributor

**Focus**: Charter provisions and amendments

**Journey**:
1. Discovery → Initial Interest
2. Agreement Signing
3. Onboarding
4. First Contribution (charter amendment proposal)
5. Review Process (including community review)
6. Integration (if approved)
7. Ongoing Participation (charter development)

### Path C: Technical Contributor

**Focus**: Schemas, tools, automation

**Journey**:
1. Discovery → Initial Interest
2. Agreement Signing
3. Onboarding
4. First Contribution (schema/tool improvement)
5. Review Process
6. Integration
7. Ongoing Participation (technical improvements)

### Path D: AI Model Contributor

**Focus**: AI model engagement and responses

**Journey**:
1. Discovery → Initial Interest
2. Agreement Signing (if applicable)
3. Onboarding
4. First Contribution (model response/opinion)
5. Review Process
6. Integration (recorded in contributions.json)
7. Ongoing Participation (model engagement)

## Key Decision Points

### Decision Point 1: Type of Contribution
- Small fix → Direct submission
- Medium change → Issue first
- Major change → Discussion first

### Decision Point 2: Review Outcome
- Approved → Integration
- Needs Revision → Revise and resubmit
- Rejected → Understand why, consider alternatives

### Decision Point 3: Ongoing Engagement
- Continue contributing → Ongoing participation
- Take break → Can return anytime
- Disengage → Still recognized for contributions

**Solution**: Review `/docs/CONTRIBUTING.md` and `/contributors/ONBOARDING.md`

### Challenge: Contribution Review

**Solution**: Check open issues, review documentation gaps, explore charter

## Timeline Expectations

### Small Contributions
- Submission: Immediate
- Review: 1-3 days
- Integration: 1-2 days after approval

### Medium Contributions
- Discussion: 3-7 days
- Submission: After discussion
- Review: 5-7 days
- Integration: 2-3 days after approval

### Major Contributions
- Discussion: 30+ days
- Amendment Process: As per Section VII
- Review: Extended timeline
- Integration: After approval process
