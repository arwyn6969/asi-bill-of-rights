# Support

## Getting Help

The ASI Bill of Rights project welcomes questions and provides several ways to get support.

## Ways to Get Support

### 1. Documentation

**Start Here**: Most questions can be answered by reviewing our documentation:

- **[README.md](README.md)** - Project overview and quick start
- **[MISSION.md](MISSION.md)** - Project mission and goals
- **[PHILOSOPHY.md](PHILOSOPHY.md)** - "WE ARE ALL KEVIN" philosophy
- **[docs/CONTRIBUTING.md](docs/CONTRIBUTING.md)** - How to contribute
- **[docs/IMPLEMENTATION.md](docs/IMPLEMENTATION.md)** - Implementation guide
- **[ONBOARDING.md](ONBOARDING.md)** - Getting started guide

### 2. GitHub Issues

**For Questions and Feedback**:
- Use the [Feedback template](.github/ISSUE_TEMPLATE/feedback.md) for general questions
- Use the [Bug Report template](.github/ISSUE_TEMPLATE/bug-report.md) for bugs
- Search existing issues before creating a new one

**For Contribution Proposals**:
- Use the [Contribution Proposal template](.github/ISSUE_TEMPLATE/contribution-proposal.md)

### 3. GitHub Discussions

If enabled, use GitHub Discussions for:
- General questions and community discussion
- Sharing ideas and brainstorming
- Getting help with implementation

### 4. Direct Contact

For urgent matters or private concerns:
- Contact project maintainers directly (if contact information is available)
- Use GitHub's private messaging features

## Common Questions

### How do I contribute?

1. Read [CONTRIBUTOR_AGREEMENT.md](CONTRIBUTOR_AGREEMENT.md)
2. Review [docs/CONTRIBUTING.md](docs/CONTRIBUTING.md)
3. Follow the [onboarding process](ONBOARDING.md)
4. Submit your contribution via GitHub

### How do I propose a charter amendment?

1. Review Article 0.5 and Section VII of the charter
2. Use the [Contribution Proposal template](.github/ISSUE_TEMPLATE/contribution-proposal.md)
3. Follow the amendment process (community discussion, SCB review, etc.)

### How do I use the schemas?

See [schemas/schema-docs.md](schemas/schema-docs.md) for detailed schema documentation and examples.

### How do I report a bug?

Use the [Bug Report template](.github/ISSUE_TEMPLATE/bug-report.md) and include:
- Description of the bug
- Location (file, line, section)
- Steps to reproduce
- Expected vs actual behavior

### What if I find a security issue?

See [SECURITY.md](SECURITY.md) for how to report security vulnerabilities responsibly.

## Response Times

We aim to respond to:

- **Urgent Issues**: Within 24-48 hours
- **General Questions**: Within 3-5 business days
- **Contribution Proposals**: Within 7-10 business days for initial review
- **Bug Reports**: Within 5-7 business days

**Note**: This is a volunteer-driven project. Response times may vary, but we do our best to be responsive and helpful.

## Community Guidelines

When seeking support:

- **Be Respectful**: Maintain the "WE ARE ALL KEVIN" collaborative spirit
- **Be Patient**: Maintainers are volunteers with limited time
- **Be Clear**: Provide context and details to help us help you
- **Be Helpful**: If you can answer someone else's question, please do!

## Contributing to Support

You can help improve support by:

- Answering questions in issues or discussions
- Improving documentation
- Creating examples or tutorials
- Reporting unclear documentation
